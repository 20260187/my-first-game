## AI에게 물어본 질문

### 질문 1: "Thonny IDE에서 가상환경(venv)을 설정하고 외부 라이브러리를 설치하려면 어떻게 해야 하나요?"
* **답변 요약:** Thonny는 내부에 자체 파이썬 패키지 관리 기능을 제공합니다. `Tools` > `Manage packages...` 메뉴를 통해 가상환경을 복잡하게 잡지 않고도 쉽게 라이브러리를 검색하고 설치할 수 있다는 안내를 받았습니다.

### 질문 2: "git push를 할 때 'rejected - non-fast-forward' 에러가 발생하는 원인과 해결 방법은 무엇인가요?"
* **답변 요약:** 원격 저장소에 로컬에는 없는 새로운 커밋이 있을 때 발생하는 에러입니다. `git pull`을 먼저 수행하여 원격의 변경 사항을 로컬과 병합한 뒤에 다시 push해야 한다는 해결책을 얻었습니다.

### 질문 3: "git commit 메시지를 이미 작성해서 완료했는데, 오타가 있어서 수정하고 싶을 때는 어떤 명령어를 쓰나요?"
* **답변 요약:** 가장 최근에 작성한 커밋 메시지라면 `git commit --amend` 명령어를 사용하여 수정할 수 있다는 답변을 확인했습니다.
## 시행착오 및 해결 과정

### 1. Thonny 설치 및 실행 오류
* **문제 상황:** Thonny IDE 설치 후 실행하는 과정에서 특정 환경 변수 꼬임으로 인해 파이썬 인터프리터가 정상적으로 연결되지 않는 문제가 발생했습니다.
* **해결 과정:** 설정을 완전히 초기화하기 위해 Thonny를 삭제 후 재설치하였으며, 내장 인터프리터(Bundled Python)를 기본값으로 지정하여 정상 구동을 확인했습니다.

### 2. Git Pull 충돌 (Conflict) 발생
* **문제 상황:** Git 명령어 실습 중 원격 저장소의 내용과 로컬 저장소의 파일 내용이 같은 라인에서 달라 `git pull` 시 충돌(Merge Conflict)이 발생했습니다.
* **해결 과정:** 코드 에디터에서 충돌이 일어난 표시(`<<<<<<<`, `=======`, `>>>>>>>`)를 확인한 후, 어떤 코드를 남길지 직접 수정하였습니다. 이후 `git add`와 `git commit`을 다시 진행하여 병합을 완료했습니다.

## 배운 점
* **Thonny IDE 활용:** 가볍고 직관적인 인터페이스를 가진 Thonny를 통해 파이썬 스크립트를 빠르게 작성하고 실행하는 환경을 익혔습니다.
* **Git 명령어의 흐름:** `add`, `commit`, `push`, `pull` 등 형상 관리의 가장 기본적이면서도 중요한 흐름을 직접 실습하며 파일의 상태 변화를 명확히 이해하게 되었습니다. 충돌이 발생하더라도 당황하지 않고 코드를 비교하여 해결하는 방법을 배웠습니다.
